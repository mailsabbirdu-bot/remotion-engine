# 🔄 Re-Scout Engine Colab Guide

Run the Re-Scout Engine in Google Colab to re-scout specific scenes with custom keyword density.

## 📁 Prerequisites (Google Drive)
The engine assumes you already have:
- `Counterism_Studio_V4/audio/` containing your `.wav` and `.txt` files.
- `Counterism_Studio_V4/manifests/production_plan.json` already generated by the main engine.

Re-scouted scenes will be saved to:
- `Counterism_Studio_V4/audio/re_scout/`

---

Copy and paste the following into a Google Colab code cell:

## 📝 How it works
1. **Interactive Loop**: The terminal will ask you for the **Scene Number** (e.g., 1, 2, 5).
2. **Scouting Modes**:
   - **More specific**: Creates denser search queries for very targeted assets.
   - **More generic**: Uses broad keywords for a more atmospheric/general look.
   - **Blank**: Uses the default extraction logic.
3. **Re-Rendering**: The engine re-downloads assets, re-audits them, and renders the scene with the same settings (padding, audio sync) as the original.
4. **Target Folder**: Output is saved in `Drive > Counterism_Studio_V4 > audio > re_scout`.
